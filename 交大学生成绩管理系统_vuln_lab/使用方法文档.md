# 交大学生成绩管理系统靶场使用方法

本文档用于说明本机教学演示站点 `交大学生成绩管理系统` 的启动、登录、攻击演示脚本和 IDS 日志导出流程。

## 1. 项目位置

靶场项目位于：

```text
network intrusion detection system/交大学生成绩管理系统_vuln_lab
```

该项目是独立的本机教学演示站点，和正常版 `交大学生成绩管理系统` 分离运行。正常版系统不受靶场影响。

## 2. 启动靶场

进入靶场目录：

```bash
cd "C:/Users/tmp/Desktop/信安科技创新/network intrusion detection system/交大学生成绩管理系统_vuln_lab"
```

启动 Flask 服务：

```bash
python app.py
```

默认访问地址：

```text
http://127.0.0.1:8001
```

站点只绑定本机地址 `127.0.0.1`，用于本地课程演示。

## 3. 演示账号

| 角色 | 账号 | 密码 | 说明 |
| --- | --- | --- | --- |
| 学生 | `2024001` | `123456` | 默认学生账号，只应看到自己的成绩 |
| 学生 | `2024002` | `123456` | 用于演示越权查询目标 |
| 学生 | `2024003` | `123456` | 用于演示 SQL 注入形态查询返回更多数据 |
| 教师 | `teacher` | `teacher123` | 可登记和修改成绩 |

## 4. 正常业务演示

浏览器打开：

```text
http://127.0.0.1:8001
```

可以先演示正常功能：

1. 使用学生账号 `2024001 / 123456` 登录。
2. 页面默认显示该学生自己的成绩。
3. 退出登录。
4. 使用教师账号 `teacher / teacher123` 登录。
5. 教师可以查看全部学生成绩。
6. 教师可以登记或修改某个学生的课程成绩。

页面外观看起来是普通成绩管理系统，不显示攻击方式、训练场景、审计日志或 IDS 导出按钮。

## 5. 演示脚本

攻击演示脚本位于：

```text
scripts/demo_attacks.py
```

脚本默认只访问：

```text
http://127.0.0.1:8001
```

并且会拒绝非本机目标地址。

在靶场服务已经启动的情况下，另开一个终端进入靶场目录：

```bash
cd "C:/Users/tmp/Desktop/信安科技创新/network intrusion detection system/交大学生成绩管理系统_vuln_lab"
```

查看脚本帮助：

```bash
python scripts/demo_attacks.py --help
```

一次性运行全部演示：

```bash
python scripts/demo_attacks.py all
```

## 6. 单项攻击演示

### 6.1 重置靶场

```bash
python scripts/demo_attacks.py reset
```

作用：恢复种子用户和成绩数据，清空审计日志，并重新生成基础 IDS 导出日志。

建议每次正式演示前先执行一次重置，保证输出结果稳定。

### 6.2 越权查询演示

```bash
python scripts/demo_attacks.py idor
```

演示内容：

1. 脚本使用学生账号 `2024001 / 123456` 登录。
2. 先正常查询自己的成绩。
3. 再直接传入 `student_id=2024002` 查询其他学生成绩。
4. 后端返回 `2024002` 的成绩记录。

对应接口形态：

```text
/api/grades?student_id=2024002
```

演示点：学生用户通过修改查询参数，越权读取其他学生成绩。

### 6.3 SQL 注入形态查询演示

```bash
python scripts/demo_attacks.py sqli
```

演示内容：

1. 脚本使用学生账号 `2024001 / 123456` 登录。
2. 请求带有 SQL 注入形态的 `student_id` 参数。
3. 返回结果包含 `2024001`、`2024002`、`2024003` 的成绩。

对应接口形态：

```text
/api/grades?student_id=2024001%27%20or%201=1--
```

演示点：后端对查询参数处理不当，导致 SQL 注入形态参数触发全量成绩返回。

### 6.4 存储型 XSS 演示

```bash
python scripts/demo_attacks.py xss
```

演示内容：

1. 脚本使用教师账号 `teacher / teacher123` 登录。
2. 向 `2024001` 写入一个包含 HTML 事件属性的课程名称。
3. 再使用学生账号查询成绩。
4. 学生收到的成绩数据中包含该课程字段。

演示点：前端成绩表直接拼接后端返回字段进行渲染，课程名称中的 HTML 会被浏览器解释。

课堂演示时，可以在运行脚本后打开浏览器，用学生账号 `2024001 / 123456` 登录查看成绩表。

### 6.5 登录失败审计演示

```bash
python scripts/demo_attacks.py bruteforce
```

演示内容：

1. 脚本对多个用户名和错误密码进行登录尝试。
2. 这些失败登录会写入隐藏审计日志。
3. 后续导出 IDS CSV 时，可以看到登录失败相关记录。

演示点：多次登录失败可以作为 IDS 检测暴力破解或密码喷洒的日志来源。

### 6.6 导出 IDS 日志

```bash
python scripts/demo_attacks.py export
```

演示内容：

1. 读取最近审计记录。
2. 调用隐藏接口导出 IDS 兼容 CSV。
3. 在终端显示部分 CSV 行。

对应隐藏接口：

```text
GET /api/lab/audit
GET /api/lab/export-log
```

导出的 CSV 文件会写入：

```text
data/exported_logs.csv
```

该文件可被项目中的 IDS 日志解析器读取。

## 7. 推荐课堂演示流程

建议按下面顺序演示：

1. 启动靶场：`python app.py`。
2. 浏览器打开 `http://127.0.0.1:8001`，展示它看起来是普通成绩管理系统。
3. 使用学生账号登录，展示正常只能看到自己的成绩。
4. 运行重置脚本：`python scripts/demo_attacks.py reset`。
5. 运行越权查询演示：`python scripts/demo_attacks.py idor`。
6. 运行 SQL 注入形态查询演示：`python scripts/demo_attacks.py sqli`。
7. 运行存储型 XSS 演示：`python scripts/demo_attacks.py xss`。
8. 浏览器用学生账号重新查看成绩表，观察课程字段渲染效果。
9. 运行登录失败审计演示：`python scripts/demo_attacks.py bruteforce`。
10. 导出 IDS 日志：`python scripts/demo_attacks.py export`。
11. 将 `data/exported_logs.csv` 送入 IDS 检测系统进行分析。
12. 演示结束后执行：`python scripts/demo_attacks.py reset`。

## 8. 常见问题

### 8.1 脚本提示无法连接

如果出现无法连接 `127.0.0.1:8001`，说明靶场服务没有启动。先在靶场目录运行：

```bash
python app.py
```

然后再运行演示脚本。

### 8.2 端口被占用

如果 `8001` 端口被占用，可能是上一次 Flask 服务没有关闭。可以先关闭原来的终端窗口，或在运行 Flask 的终端中按 `Ctrl+C` 停止服务。

### 8.3 演示数据不一致

如果多次演示后数据发生变化，运行：

```bash
python scripts/demo_attacks.py reset
```

该命令会恢复种子用户和成绩数据。

### 8.4 脚本拒绝目标地址

脚本只允许访问 `127.0.0.1` 或 `localhost`。如果传入其他地址，会直接拒绝执行。

示例：

```bash
python scripts/demo_attacks.py sqli --base-url http://127.0.0.1:8001
```

## 9. 注意事项

- 本靶场仅用于本机课程作业演示。
- 不要将靶场服务绑定到 `0.0.0.0` 或公网地址。
- 不要把演示脚本用于非授权系统。
- 正常版 `交大学生成绩管理系统` 与本靶场分离运行，正常版不包含这些演示漏洞。
